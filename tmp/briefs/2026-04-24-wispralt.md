# Brief: WisprAlt — Local/Self-Hosted Wispr Flow Replacement

## Why

User pays $15/mo for Wispr Flow on macOS for FN-key push-to-talk dictation. Wispr Flow is cloud-only — audio + periodic screenshots are sent to OpenAI/Meta servers, which triggered a privacy backlash in early 2026. User wants a self-hosted equivalent that:

1. Recreates Wispr Flow's FN-key push-to-talk dictation UX with sub-500ms total latency.
2. Adds a **meeting recorder** mode (not in Wispr Flow) for capturing in-person and remote meetings with high-quality transcription and speaker labels — for business use.
3. Runs server-side on user's existing always-on M4 Mac mini (16GB), so client devices stay lightweight and the same backend serves multiple devices/friends.
4. Ships as a polished GitHub project with one-step setup (ideally via Claude Code).

## Context

- **User's hardware**: MacBook Air M4 (client, primary), M4 Mac mini 16GB (server, always-on, has spare capacity).
- **User's domains**: Owns multiple domains already on Cloudflare. Will use a subdomain (e.g. `transcribe.<existing-domain>`) for the API endpoint.
- **Language preference**: English-only is fine for both modes.
- **Permission tolerance**: User accepts one-time setup steps (Accessibility, Input Monitoring, Microphone, Screen Recording on client; cloudflared install + token on server). Does NOT want any client-side network configuration (no Tailscale, no VPN, no LAN-only).
- **Existing repo state**: `/Users/omidzahrai/Desktop/CODEBASES/TOOLS/wisprflowALT/` is empty (only a single first commit). Greenfield.

### Research findings (key points)

- **Dictation model — Parakeet TDT 0.6B v2 (NVIDIA, MLX/CoreML port)**: 1.69% WER on LibriSpeech, 110-300x realtime on Apple Silicon, ~150ms typical for 10s clip. English-only RNN-T. Perfect for short, fast, sloppy speech.
- **Meeting model — CrisperWhisper (Whisper-large-v3 fine-tune) + WhisperX + Pyannote 3.1 + DeepFilterNet 3**: Outperforms vanilla Whisper-large-v3 on AMI meeting benchmark, fewer hallucinations on silence/crosstalk. Combined ~3GB RAM, runs on MPS.
- **FN/Globe key detection**: macOS `CGEventTap` listening for `NSFlagsChanged`, checking `NX_SECONDARYFNMASK` (0x800000) or keycode 63. Native Swift only — Python/Electron has unacceptable latency and reliability issues here.
- **Text injection**: Try `AXUIElement` first (fast, no clipboard interference), fall back to clipboard + synthetic Cmd+V (universal, works in Electron/terminals). Restore clipboard 200ms after paste.
- **System audio capture**: ScreenCaptureKit (macOS 13+) supports separate mic + system audio channels natively. No BlackHole/Loopback/kernel extension required. Requires Screen Recording permission.
- **Networking — Cloudflare Tunnel**: Free, zero port forwarding, persistent custom HTTPS URL, ~50-200ms overhead same-region. macOS install: `brew install cloudflared` + `sudo cloudflared service install <token>`. Auto-restarts via launchd. Reference project [OpenSuperWhisper](https://github.com/Starmel/OpenSuperWhisper) for Swift FN-key scaffolding (study, don't fork — building clean for repo polish).

## Decisions

### Architecture
- **Server-only on Mac mini** — user does not want fully-local mode. Single backend, multi-client.
- **Two endpoints, two models, both resident in memory:**
  - `POST /transcribe/dictate` → Parakeet TDT 0.6B v2 (MLX), optimized for latency.
  - `POST /transcribe/meeting` → DeepFilterNet 3 → WhisperX with CrisperWhisper weights → Pyannote 3.1 diarization, optimized for quality. Async/long-running OK.
- **Auth**: Bearer token (32+ random bytes) in `Authorization` header. App-layer validation. No Cloudflare Access (friends just paste a key).

### Networking
- **Cloudflare Tunnel** with subdomain off existing domain. — Why: zero client-side setup, no port forwarding, free, stable URL, friends install nothing. — How to apply: setup script registers `cloudflared` as launchd service; user picks subdomain via Cloudflare dashboard.
- No Tailscale, no LAN-only mode, no port forwarding.

### Dictation client (FN-key push-to-talk)
- **Swift native macOS app** (SwiftUI + AppKit menubar). — Why: only path that hits the latency target with reliable FN-key handling and clean text injection. — How to apply: distributed as a signed `.app` bundle; user double-clicks to install.
- FN/Globe key hold-to-record via `CGEventTap` + `NSFlagsChanged`.
- Mic capture via AVFoundation, ring buffer, stop on key release, upload to `/transcribe/dictate`.
- Text injection: AXUIElement first, clipboard+Cmd+V fallback.
- **Triple-tap of FN within ~400ms total** = start/stop meeting recording. Tight window prevents false triggers.

### Meeting recorder
- **Menubar Record button** + **FN triple-tap** both trigger recording.
- **Dual-channel capture via ScreenCaptureKit**: mic on channel 1, system audio on channel 2, **separate channels not mixed**, timestamps preserved. This is the revolutionary part.
- **Server-side processing**: each channel transcribed independently with WhisperX+CrisperWhisper. Merge by timestamp.
  - Channel 1 lines → labeled "You" by default.
  - Channel 2 lines → labeled "Other" (or further split via Pyannote 3.1 into "Other 1, 2…" if multiple remote speakers).
- **In-person mode auto-detection**: if channel 2 (system audio) is silent for the whole recording, switch to single-channel mode — mic only, Pyannote 3.1 diarization splits speakers as "Speaker 1, 2, …". Menubar UI shows current mode ("Meeting" vs "In-person").
- **Always capture system audio** — auto-detection determines mode, no user toggle needed.
- **Output**: JSON (canonical, with speakers + word-level timestamps) + SRT + TXT. Saved to `~/Documents/WisprAlt/Meetings/` with filename `YYYY-MM-DD_HHMM_<title>.{ext}`. Title defaults to "meeting" or "in-person".
- **Speaker rename UI**: small menubar window listing past transcripts. Click → rename speakers in place. Defaults are "You", "Other", "Speaker N" — never required to rename.
- **Notification when transcription complete**.

### Setup experience
- **Server (Mac mini)**: single setup script (`./scripts/setup-server.sh`). Installs Python deps, downloads Parakeet + CrisperWhisper + Pyannote + DeepFilterNet weights, installs cloudflared, generates API key, prints copy-paste client config snippet.
- **Client (any Mac)**: download signed `.app`, double-click, paste server URL + API key into settings on first launch, grant 4 permissions (Accessibility, Input Monitoring, Microphone, Screen Recording). Done.
- **Claude Code-friendly**: `.claude/commands/` includes `/setup-server`, `/setup-client`, `/test-connection` so users can run `claude` in a fresh checkout and have everything wired up by conversation.

## Rejected Alternatives

- **Tailscale / ZeroTier / LAN-only** — User explicitly rejected anything requiring client-side network setup. Cloudflare Tunnel beats it on zero-client-config goal.
- **Port forwarding + DuckDNS + Let's Encrypt** — Lowest latency but ongoing maintenance (ISP IP changes, router resets, CGNAT risks). Not worth it.
- **Fully-local mode on MacBook Air** — User chose server-only. Removing the dual-mode complexity simplifies repo significantly.
- **Python or Electron client** — Python+pyobjc has interpreter startup latency and unreliable FN-key/text-injection paths. Electron is heavier and worse at native macOS APIs. Swift wins decisively.
- **Forking OpenSuperWhisper** — Saves time but produces a derivative repo. Building clean keeps the project polished and easier to set up.
- **Whisper-large-v3-turbo for dictation** — ~2x slower than Parakeet for English-only short clips. Parakeet wins on the dictation latency target.
- **NVIDIA Canary-Qwen 2.5B for meetings** — Best WER on benchmarks but designed for NVIDIA GPUs, not natively optimized for Apple Silicon. CrisperWhisper is the practical Mac winner.
- **NeMo MSDD diarization** — Better on NVIDIA, worse on Mac than Pyannote 3.1. Pyannote integrates cleanly with WhisperX.
- **Mixed mic+system audio capture** — Loses the "who's speaking" signal. Dual-channel is strictly better.
- **Cloudflare Access (email allowlist)** — User said friend sharing is not important. Bearer token alone is sufficient for stated threat model.
- **Toggle-mode dictation** — Out of scope for v1. Push-to-talk only.
- **LLM cleanup of dictation output** — Out of scope for v1. Plain transcription only.

## Direction

Build a two-component repo: (1) a Swift native macOS menubar client handling FN hold-to-record dictation, FN triple-tap meeting recording with dual-channel ScreenCaptureKit capture, and text injection; and (2) a Python FastAPI server on the M4 Mac mini exposing two endpoints — Parakeet for low-latency dictation and CrisperWhisper+Pyannote+DeepFilterNet for high-quality meeting transcription — exposed publicly via Cloudflare Tunnel on a subdomain of the user's existing domain, secured by a single bearer-token. Setup is one script on the server, one signed `.app` on the client, no networking configuration required by end users. Repo includes Claude Code commands so the entire setup can be driven via conversation in a fresh checkout.
